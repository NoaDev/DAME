Editing Sprite Trails in DAME

With DAME you can lay out a path to automatically display a sequence of sprites along it that will be updated whenever you make a change to a path or the sequence.

  1. Setting Up
  2. Editing

1. Setting Up

The first thing you must do is to lay out a path. With the path ready (you can adjust it later), right click on the path and choose Attach Sprite Trail from the context menu. Then all you need to do is move to the sprite layer you wish to place the sprite trail in and left click. This will display the Sprite Trails window, if it wasn't open already:

Unlike the screenshot, it's likely that the window will not have any sprites associated with it, so you'll need to set it up. You can click the Add From Current Sprite Entry button to bring the sprite entry currently selected in the sprites tab to the panel. As long as the sprite trail object or the path are selected the trail should immediately refresh with that sprite displayed at periodic intervals along the path.

2. Editing

Firstly, if the sprite trail window is not open you can bring it up by going to the Tools menu and selecting Show Sprite Trail Settings.

Sprite Trails will always automatically refresh as long as they are associated with a selected sprite trail object or path with a sprite trail attached. Whenever you want to edit a sprite trail you must select that sprite trail and click Get From Selected Trail before modifying any values. You can send that data over to another sprite trail by clicking Send To Selected Trail.

There are only a few options controlling sprite trails but they allow for many different variations as you can see from this screenshot:

It is possible to have multiple occurrences of the same sprite in the sequence. You can also modify each entry to control the region displayed within the sprite. The X, Y, Wid and Ht values control this and modifying them will change only the selected sprite in the display at the top.

You can also control the anchor of the selected sprite in the list with the Anchor X and Anchor Y values. This will change the position where the sprite will attach itself to the path.

Normally the sprites are spaced based on the Distance Between Sprites value. However, there is a problem with this: If the path length is not an exact multiple of the distance then the last sprite will not necessarily be at the end. This is even more evident with closed paths as the beginning and end sprites may end up bunched together. Using the Clever Spread Algorithm will ensure that the distance is only used as a guideline and the sprites will be spaced much more intelligently.

You can make the sprites rotate along with the path using Rotation Follows Path and you can offset them by an angle with the Rotation Offset, whether they follow the paths rotation or not.

Finally, the sprite sequence can be controlled. You have options allowing you to display the sprites in the sequence as listed or to be random, controlled by a Random Seed. You can also choose to have the first and last sprite in the list be the first and last sprites in along the path, along you to 'close' a trail properly.
